Replace the placeholder src/scene.rs with the full Scene implementation.

Scene stores: width (u32), height (u32), fps (u32), duration_secs (f64), and a fontdb (initialized once at construction). The fontdb is NOT per-frame.

Per locked decisions:
- Constructor: `Scene::new(width, height, fps)` — video config set at scene construction
- Render API: closure pattern `scene.render(|s| { s.add(circle); }, path)`
- Error type: `Result<(), EidosError>` — single opaque type

Implement Scene::new() with eager validation:
1. Width and height must both be even (H.264 requirement). If odd, return `Err(EidosError::InvalidConfig("width must be an even number".into()))` and same for height.
2. fps must be > 0. If 0, return `Err(EidosError::InvalidConfig("fps must be greater than zero".into()))`.
3. Probe for ffmpeg on PATH: run `Command::new("ffmpeg").arg("-version").output()`. If `Err(NotFound)`, return `Err(EidosError::RenderFailed("ffmpeg not found on PATH — install ffmpeg to use eidos".into()))`.
4. Initialize fontdb: `fontdb::Database::new()`, call `db.load_font_data(ttf_noto_sans::REGULAR.to_vec())`. Store in Scene.
5. Default duration: 1 second. Add `Scene::duration(mut self, secs: f64) -> Self` method for configuration.

Implement the render closure approach. The closure receives `&mut SceneBuilder` (an inner struct that accumulates primitives). After the closure runs, SceneBuilder's primitives list is used to generate the video.

```rust
pub struct Scene {
    width: u32,
    height: u32,
    fps: u32,
    duration_secs: f64,
    fontdb: resvg::usvg::fontdb::Database,
}

pub struct SceneBuilder {
    primitives: Vec<crate::primitives::Primitive>,
}

impl SceneBuilder {
    pub fn add(&mut self, p: impl Into<crate::primitives::Primitive>) -> &mut Self {
        self.primitives.push(p.into());
        self
    }
}

impl Scene {
    pub fn new(width: u32, height: u32, fps: u32) -> Result<Self, EidosError> { ... }

    pub fn duration(mut self, secs: f64) -> Self {
        self.duration_secs = secs;
        self
    }

    pub fn render<F, P>(&self, build_scene: F, output_path: P) -> Result<(), EidosError>
    where
        F: Fn(&mut SceneBuilder),
        P: AsRef<std::path::Path>,
    {
        let mut builder = SceneBuilder { primitives: Vec::new() };
        build_scene(&mut builder);

        // Generate SVG from builder.primitives
        let svg_string = crate::svg_gen::build_svg_document(
            self.width, self.height, &builder.primitives
        );

        // Rasterize once (static scene — all frames identical)
        let bgra_frame = crate::svg_gen::rasterize_frame(
            &svg_string, self.width, self.height, &self.fontdb
        )?;

        // Calculate total frames
        let total_frames = (self.fps as f64 * self.duration_secs).round() as u64;

        // Encode via ffmpeg
        crate::svg_gen::encode_to_mp4(
            &bgra_frame, total_frames, self.width, self.height, self.fps,
            output_path.as_ref()
        )
    }
}
```

For Phase 1 (static scene), the same bgra_frame bytes are written to ffmpeg stdin `total_frames` times. This is intentional and noted with a comment: "Phase 2 will call build_scene() per-frame with animated state instead."

Use `resvg::usvg::fontdb` — it is re-exported via resvg, so `use resvg::usvg::fontdb;` in Cargo (no separate fontdb dependency needed).

**build_svg_document(width, height, primitives)**

Builds an `svg::Document` with:
1. Set `width`, `height`, `viewBox` to `(0, 0, width, height)` — all three must match for pixel-exact positioning (avoids Pitfall 6 from research).
2. Add black background rectangle: `Rectangle::new().set("x", 0).set("y", 0).set("width", width).set("height", height).set("fill", "black")`.
3. Collect all SVG Definitions first (needed for arrow markers). Loop through primitives, if a primitive produces defs (only Arrow does), add the Definitions node BEFORE adding shape nodes. This avoids Pitfall 3 (defs must precede references).
4. Loop through primitives again, add shape nodes.
5. Return `doc.to_string()`.

The primitives dispatch uses `match` on the `Primitive` enum (defined in src/primitives/mod.rs). In this plan, the match arms for each primitive variant call conversion functions that will be fully implemented in Plan 01-03 and 01-04. For now, stub each arm to return an empty Group: `Primitive::Circle(_) => doc` etc. Plan 01-05 will complete the dispatch. Leave a TODO comment on each stubbed arm.

**rasterize_frame(svg_str, width, height, fontdb)**

```rust
pub fn rasterize_frame(
    svg_str: &str,
    width: u32,
    height: u32,
    fontdb: &resvg::usvg::fontdb::Database,
) -> Result<Vec<u8>, EidosError> {
    use resvg::usvg::{self, Options};
    use resvg::tiny_skia;

    let options = Options::default();
    let tree = usvg::Tree::from_str(svg_str, &options, fontdb)
        .map_err(|e| EidosError::RenderFailed(format!("SVG parse error: {}", e)))?;

    let mut pixmap = tiny_skia::Pixmap::new(width, height)
        .ok_or_else(|| EidosError::RenderFailed("failed to allocate pixmap".into()))?;

    resvg::render(&tree, tiny_skia::Transform::identity(), &mut pixmap.as_mut());

    Ok(pixmap.data().to_vec())  // BGRA8 bytes
}
```

CRITICAL: The fontdb must already have Noto Sans loaded (done in Scene::new()). Do NOT create a new fontdb here — accept it as a parameter to avoid per-frame font loading.

**encode_to_mp4(bgra_frame, total_frames, width, height, fps, output_path)**

```rust
pub fn encode_to_mp4(
    bgra_frame: &[u8],
    total_frames: u64,
    width: u32,
    height: u32,
    fps: u32,
    output_path: &std::path::Path,
) -> Result<(), EidosError> {
    use std::process::{Command, Stdio};
    use std::io::Write;

    let mut child = Command::new("ffmpeg")
        .args([
            "-f", "rawvideo",
            "-pix_fmt", "bgra",          // matches tiny-skia Pixmap::data() — NOT rgba (Pitfall 2)
            "-s", &format!("{}x{}", width, height),
            "-r", &fps.to_string(),
            "-i", "pipe:0",
            "-pix_fmt", "yuv420p",       // H.264 requires YUV
            "-c:v", "libx264",
            "-y",                        // overwrite output without prompting
            output_path.to_str()
                .ok_or_else(|| EidosError::InvalidConfig("output path is not valid UTF-8".into()))?,
        ])
        .stdin(Stdio::piped())
        .stderr(Stdio::null())           // suppress ffmpeg log noise; use Stdio::inherit() for debugging
        .spawn()
        .map_err(|e| EidosError::RenderFailed(format!("failed to spawn ffmpeg: {}", e)))?;

    let stdin = child.stdin.as_mut()
        .ok_or_else(|| EidosError::RenderFailed("failed to open ffmpeg stdin".into()))?;

    // Write the same static frame total_frames times
    // Phase 2 will instead write a different frame for each animation step
    for _ in 0..total_frames {
        stdin.write_all(bgra_frame)
            .map_err(|e| EidosError::RenderFailed(format!("failed to write frame to ffmpeg: {}", e)))?;
    }

    drop(child.stdin.take()); // close stdin so ffmpeg knows input is done

    let status = child.wait()
        .map_err(|e| EidosError::RenderFailed(format!("ffmpeg wait failed: {}", e)))?;

    if !status.success() {
        return Err(EidosError::RenderFailed(
            format!("ffmpeg exited with non-zero status: {}", status)
        ));
    }

    Ok(())
}
```

Note: `drop(child.stdin.take())` is required before `child.wait()` — ffmpeg will block waiting for more input if stdin is not closed. This is a common gotcha when piping to subprocesses.
