### Integration Gap: INT-01

`Axes::to_primitives()` resolves internal coordinate bounds from Heckbert tick generation — `x_axis_min/max` are the first/last tick values, not the raw `AxisRange` values. `SplineFit::to_bezier()` requires the caller to supply `visual_pts` pre-mapped to pixel space using the same bounds.

No public method on `Axes` exposes the tick-adjusted bounds. The `gam_plot.rs` example and `gam_viz.rs` tests hardcode `map_x_ex`/`map_y_ex` functions using the raw explicit range. This happens to be correct for the specific range `[0,10]/[-1.4,1.4]` (where ticks align exactly with the explicit bounds), but would produce spatial misalignment between the `SplineFit` bezier and the `Axes` grid for ranges where tick generation extends past the explicit min/max.

**Remediation:** Add `pub fn plot_bounds(&self) -> (f64, f64, f64, f64)` to `Axes` returning tick-adjusted `(x_min, x_max, y_min, y_max)`.

**Affected requirement:** GAM-02 (partial — works for current use case, fragile for general use).
